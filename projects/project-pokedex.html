Project Pokedex

tldr; REST API web app for .

tools: Asynchronous JSON, REST API, HTML&CSS, JavaScript, JQuery

Our final project in New York Code + Design Academy's Front-End 101 bootcamp was to use some kind of open-source API to make something. Very open-ended. I wanted to do something that was fun and nerdy, but that I could also continue to build off of in the future.

Thank goodness I found an enormous Pokemon API.

The API is so thorough and detailed you could spend hours just pouring through information, so I wanted to make sure my project was narrow enough in scope that I could actually accomplish it in time. I decided on a few key features: given an input ID, dynamically generate a card image for that pokemon and when you click it, show it's entire evolution family.

The markup was easy — I don't know that I'll ever get sick of flexbox. I love it so much.

Then I hit my first wall. I was so used to building static websites and only using Javascript for DOM manipulations that I didn't really know where to start mapping which endpoints I would hit and when.

I'm lucky enough to work alongside talented developers who are also generous with their time, and one of them showed how he likes to think through architecture before even starting to code.

I always thought the whole drawing lots of pictures thing was kind of an exaggeration...turns out it's incredibly helpful.

That was the end of the help — my friend then told me to do the rest myself (no matter how bad) and then he'd go over everything with me and talk about how to make it better. He assured me that debugging my own code is the most necessary skill and the best teacher.

So I set to work.

I think the hardest thing to wrap my mind around was the very concept of asynchronous data; suddenly I had to wait for response to come back. I read about callbacks and promises. I had to think about scope constantly. I spent hours in the console tracing the compilers' process trying to see why it wasn't working.

Finally I got it basically working. Sidenote: look how many Eevee evolutions there are now.

Even though I was so pleased to reach a basic level of functionality I knew there were glaring problems. The evolution cards take the longest to come back so if you click on the first pokemon before. I made a half-baked attempt at recursion that worked, but wasn't done well. The evolutions aren't always in the right order.

I presented it in class. People liked it. At least I think they did. Pokemon is still cool, right?

My favorite thing about this project though is how continuable it is. What about displaying evolution triggers or levels? Maybe if you click on a pokemon card it flips over and shows its stats and moveset. Maybe I should try and build it all over again in React (I actually think I'll do that. Seems like it's all about React these days, doesn't it?).
