Chartbeat Support Site

tldr; Designed, coded, and launched brand new support site for Chartbeat customers.

Tools Used: HTML/CSS, Bootstrap.js, Jekyll, Sketch.

The first thing I ever did—the project that really got me excited about coding and design—was a complete overhaul of the client-facing support site for Chartbeat. Before the re-design, the support site was just an list of FAQs built off of the support inbox. Not only was it a little bit outdated looking, but during my time on the Chartcorps support team, I created a ton of implementation documents and educational videos that needed to live somewhere, and ended up on a series of disconnected, static web pages, with no relationship or hierarchy.

It was hard for anyone, customer or coworker, to find specific resources on their own.

That was the problem we decided to solve. My then-coworker, now NYC designer/developer Sam Chieng, teamed up to tackle this project beginning to end: design, code, execute, and launch a fully integrated support experience.

Another Chartcorps teammate, PJ Smith, did all the JavaScript for the integration with Chartbeat’s status page. That might actually be my favorite part.

The finished product after many, many months of coding and googling things.

You can see it live at support.chartbeat.com!

We also got a chance to write about the experience and process for Chartbeat’s blog—check out part 1 (by me) and part 2 by Sam.

Would you believe each one of these took like six hours?

While we worked collaboratively throughout most of the whole project, we did divide up some of the work based on our skillsets; Sam already had a background in coding and building websites whereas I had only started teaching myself HTML & CSS only a few months previously.

So she build a ton of the foundation and site architecture (including researching static site generators and learning all about how Jekyll works), and I focused on illustrating the three images for the homepage cards. We didn’t want to have to think about copyrights and ownership so we figured we’d make our own custom graphics for the main landing page.

I like to think it's kind of R2D2 meets that Pixar lamp.

Like all initial launches, the first version of the Support Site still had some holes in it, and when Hackweek came around at Chartbeat, I decided that my project should be making a 404 page for the Support Site!

Which meant I needed to learn how to do that.

I started with the design—which of course went through many iterations—and using a combination of Sketch and Illustrator (and Photoshop to turn a bunch of layers into a gif) finally got the "robot-staring-at-a-flickering-light-bulb" live- check it out at support.chartbeat.com/foo.

Not only was this my first serious application of web design and HTML/CSS, as we hosted it within Chartbeat’s repo on Github, it started to get me familiar with version control and using git, and is responsible for my first little green "proof of commit" squares.
